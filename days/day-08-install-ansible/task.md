# Day 08 – Install Ansible Using pip3

## Task

During a discussion on automation and configuration management, the Nautilus DevOps team decided to use **Ansible** due to its simplicity and minimal setup requirements.

The jump host will act as the **Ansible controller** to manage other servers.

## Requirements

1. Install **Ansible version 4.7.0** on the jump host
2. Use **pip3 only** for installation
3. Ensure the Ansible binary is available **globally for all users**

This means any user on the system should be able to run Ansible commands.
